ESP Series Microcontrollers

Complete guide to ESP8266 and ESP32 - The powerhouse WiFi/BLE chips for IoT applications

What is ESP Series?

ESP series microcontrollers by Espressif Systems are low-cost, low-power system-on-chips (SoCs) with integrated WiFi and Bluetooth capabilities. They revolutionized IoT development by bringing wireless connectivity to microcontroller projects.

  • Integrated Wireless: Built-in WiFi (and BLE in ESP32) eliminates need for separate modules
  • High Performance: Clock speeds up to 240MHz (ESP32) with dual-core options
  • Rich Peripherals: ADCs, DACs, touch sensors, motor PWM, and more
  • Low Power Modes: Deep sleep currents as low as 10μA
  • Dual Core: ESP32 features two Xtensa LX6 cores for multitasking
  • Cost Effective: Much cheaper than WiFi/BLE modules + separate MCU
  • Community Support: Massive ecosystem of libraries and frameworks
  • Development Flexibility: Programmable with Arduino IDE, ESP-IDF, MicroPython
  • Scalable: From simple sensors to complex IoT gateways
  • Production Ready: Used in commercial products worldwide
  • Home Automation: Smart lights, switches, and sensors
  • IoT Devices: Weather stations, smart appliances
  • Wireless Sensors: Battery-powered remote monitoring
  • Robotics: WiFi/Bluetooth controlled robots
  • Industrial IoT: Equipment monitoring and control

Programming NodeMCU

NodeMCU can be programmed with Arduino IDE after adding ESP8266 support:

// Simple WiFi Web Server Example
#include <ESP8266WiFi.h>

const char* ssid = "YourSSID";
const char* password = "YourPassword";

WiFiServer server(80);

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  
  server.begin();
}

void loop() {
  WiFiClient client = server.available();
  if (client) {
    client.println("HTTP/1.1 200 OK");
    client.println("Hello from NodeMCU!");
    client.stop();
  }
}

Power Consumption

  • Active Mode: ~70mA (WiFi on)
  • Light Sleep: ~0.9mA (CPU paused)
  • Deep Sleep: ~20μA (RTC memory retained)

Advanced ESP32 Features

Dual Core Operation

ESP32 has two cores that can run tasks simultaneously:

// Run task on Core 0
xTaskCreatePinnedToCore(
    Task1,    // Task function
    "Task1",  // Name
    10000,    // Stack size
    NULL,     // Parameters
    1,        // Priority
    NULL,     // Task handle
    0         // Core 0
);

void Task1(void * pvParameters) {
    while(1) {
        // Your code here
        vTaskDelay(1000);
    }
}

Bluetooth Low Energy (BLE)

ESP32 includes both classic Bluetooth and BLE:

// Simple BLE Server Example
#include <BLEDevice.h>

BLEServer *pServer;
BLEService *pService;
BLECharacteristic *pCharacteristic;

void setup() {
  BLEDevice::init("ESP32_BLE");
  pServer = BLEDevice::createServer();
  pService = pServer->createService("1234");
  
  pCharacteristic = pService->createCharacteristic(
    "ABCD",
    BLECharacteristic::PROPERTY_READ |
    BLECharacteristic::PROPERTY_WRITE
  );
  
  pCharacteristic->setValue("Hello World");
  pService->start();
  pServer->getAdvertising()->start();
}

Ultra Low Power Techniques

ESP32 has several power saving modes:

  • Modem Sleep: WiFi/BLE off (~3mA)
  • Light Sleep: CPU paused (~0.8mA)
  • Deep Sleep: Only RTC running (~10μA)
  • Hibernation: Lowest power (~5μA)
// Deep Sleep Example
#include <esp_sleep.h>

void setup() {
  esp_sleep_enable_timer_wakeup(1000000 * 60); // 1 minute
  esp_deep_sleep_start();
}

void loop() {} // Never reached

ESP-NOW Protocol

Direct device-to-device communication without WiFi router:

// ESP-NOW Example
#include <esp_now.h>

void OnDataSent(const uint8_t *mac_addr, esp_now_send_status_t status) {
  Serial.println(status == ESP_NOW_SEND_SUCCESS ? 
    "Delivery Success" : "Delivery Fail");
}

void setup() {
  WiFi.mode(WIFI_STA);
  if (esp_now_init() != ESP_OK) {
    Serial.println("ESP-NOW Init Failed");
    return;
  }
  
  esp_now_register_send_cb(OnDataSent);
